# 네이버 API 인사이트 대시보드 고도화 작업 계획 (v1.3.x)

본 대시보드는 네이버 오픈 API를 활용하여 트렌드 및 검색 데이터를 분석하는 도구로, 최근 사용자 피드백에 따라 **종합 대시보드 신설, 우측 AI 마케팅 코필럿 도입, 세션 상태 유지(Persistence) 기능 구현, 그리고 대시보드 소개 페이지의 레이아웃 최적화** 작업을 순차적으로 완수하였습니다. 본 계획서는 지금까지의 구현 성과와 향후 추가 안정성 확보를 위한 작업 계획을 정의합니다.

## User Review Required

> [!IMPORTANT]
> **안내 대시보드(소개)의 레이아웃 단독 구동**
> '🏠 대시보드 소개' 메뉴 of 경우 AI 마케팅 코필럿 챗봇을 표출하지 않고 전체 너비(100%)로 구성되도록 하였습니다. 그 외 모든 분석 메뉴와 종합 대시보드는 13:4 비율의 2컬럼 레이아웃을 통해 우측에 AI 코필럿 챗봇 툴바를 고정 표출합니다.
> 이에 따라 화면 구조적 통일성 대비 소개 화면의 유연성에 대한 검토를 부탁드립니다.

> [!NOTE]
> **세션 데이터 영속성 (Persistence)**
> 페이지를 새로고침하거나 왼쪽 사이드바에서 탭을 전환하더라도, 사용자가 입력한 필터 설정(검색어 목록, 기간, 기기, 성별 등) 및 분석 결과 데이터프레임이 소실되지 않고 그대로 캐시되어 브라우저 메모리에 상태가 유지됩니다.

### Manual Verification
1. **메뉴 스왑에 따른 입력값 유지 확인**: '📈 검색어 트렌드 분석'에서 검색 조건을 입력하고 조회한 후, '🏠 대시보드 소개' 나 '🛒 쇼핑 검색 분석' 등 다른 메뉴로 이동했다가 돌아왔을 때 입력 폼과 차트 결과물이 그대로 유지되는지 검증합니다.
2. **대시보드 소개 화면의 전체 너비 검증**: '🏠 대시보드 소개' 메뉴에 접속했을 때 우측에 AI 코필럿용 공백 컬럼 없이 화면이 100% 꽉 차게 렌더링되는지 확인합니다.
3. **종합 대시보드 구동 검증**: 임시 예시 데이터(올라운드 가구숍)로 KPI 카드 및 차트가 완벽하게 그려지는지 보고, '📦 내 쇼핑몰 상품 진단'에서 새로운 실적 파일(.xlsx)을 로드했을 때 종합 대시보드의 데이터가 자동으로 해당 파일의 총합 지표로 갱신되는지 점검합니다.
4. **다차원 검색 키워드 개수 제한 및 탭 검증**: 다차원 분석 메뉴(쇼핑, 블로그 등)에서 키워드를 6개 입력하여 실행 시 정상적으로 5개 제한 경고가 표출되는지 테스트하고, 5개 이하 입력 시 하단 결과 테이블이 각 키워드별 탭으로 구분되어 표출되는지 확인합니다.
